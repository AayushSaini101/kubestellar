Consistent, Heterogeneous, and Scalable Edge Configuration Management

{{< blocks/link-down color="info" >}}
{{< /blocks/cover >}} {{% blocks/lead color="primary" %}} KCP-Edge is a subproject of kcp focusing on concerns arising from edge multicluster use cases: - Hierarchy, infrastructure & platform, roles & responsibilities, integration architecture, security issues - Runtime in[ter]dependence: An edge location may need to operate independently of the center and other edge locations​ - Non-namespaced objects: need general support - Cardinality of destinations: A source object may propagate to many thousands of destinations. ​ {{% /blocks/lead %}} {{< blocks/section color="dark" >}} {{% blocks/feature icon="fa-lightbulb" title="Our Mission:" %}} To develop consistent interfaces for scalable state-based configuration management of heterogeneous edge locations. {{% /blocks/feature %}} {{% blocks/feature icon="fab fa-github" title="Contributions welcome!" url="https://github.com/kcp-dev/edge-mc/blob/main/CONTRIBUTING.md" %}} View our ['good-first-issue' list](https://github.com/orgs/kcp-dev/projects/12/views/4) and create a [Pull Request](https://github.com/kcp-dev/edge-mc/blob/main/CONTRIBUTING.md) to contribute to our **GitHub**.
